# 데일리 리뷰 재구성 — 독자 순서 + 하루 요약 (2026-07-17)

## 문제

재구성 전 데일리는 AI 성공일 기준 **9개 섹션**(델타 → 장기기억 제안 → 인물 신호 →
프로젝트 신호 → 오늘 메일 핵심 → 내가 보낸 것 → 개입 필요 → 미답변 → 기한 신호)이
전부 같은 `##` + 불릿으로 나열됐다.

1. **화면 순서 = 파이프라인 순서.** 처리 단계(수확→디제스트→개입) 순서대로 쌓여서
   행동을 결정하는 '개입 필요'가 9개 중 7번째.
2. **같은 스레드가 최대 3번 반복.** 요청 건은 핵심 + 개입 + 기한 신호에, 결정 건은
   델타 + 장기기억 제안 + 핵심에 — 같은 문장이 그대로 중복되기도 했다. 섹션마다
   독립 설계라 교차 중복 제거가 없던 것이 "복잡함"의 최대 원인.
3. **'오늘 메일 핵심'이 큐레이션이 아니라 전수 나열.** '미답변'은 개입 큐와 같은
   기준으로 좁힌 뒤라 **정의상 개입 목록의 부분집합** — 순수 중복 섹션이었다.
4. **10초 스캔 불가.** 최상단 델타는 변경 로그이지 요약이 아니다.

## 설계 — 독자의 질문 순서 (9개 → 머리 요약 + 4개)

| 독자의 질문 | 섹션 | 재료 |
|---|---|---|
| 오늘 어땠나? | (맨 위, 제목 없는 문단) | AI **하루 요약** 3~5문장, 비-AI 날은 결정론 한 줄 통계 |
| 내 차례인 게 뭔가? | `## 지금 할 일 (N건)` | 개입 큐 평탄화 — 그룹 헤더 대신 항목별 마커(🔴결정·🟠회신·🟡/⚪정체) + 긴급도 칩 `[상|중|하]` + `⏰기한` 속성. 부연은 한 줄만(AI 사유·제안 우선, 없으면 「근거 인용」). 정체는 인용 생략 |
| 뭐가 정해졌나? | `## 오늘 확정·변경 (N건)` | 수확 델타 + 장기기억 제안 병합. 델타가 이미 언급한 스레드의 제안은 별도 줄 없이 말미 "※ 반영 대기 N건" 안내만. 비-AI 날은 섹션 생략 |
| 그 외 흐름은? | `## 오늘 흐름 (그 외 N건)` | 업무 스레드 중 위(할 일·확정·오늘 종결)에 **안 나온 것만** |
| 참고 | `## 참고` (웹에선 접힘) | 내가 보낸 것 · 내 회신으로 종결된 요청 · 인물/프로젝트 신호 · 잔여 기한(할 일에 ⏰로 안 붙은 것만) · 수신/제외 통계 |

- '미답변' 섹션 폐지 — 개입 목록의 부분집합이라 정보가 0. 오래된 미답변은 이미
  ⚪ 정체로 큐에 승격된다.
- '기한 신호' 독립 섹션 폐지 — 할 일 항목의 `⏰기한` 속성으로 흡수, 잔여만 참고에.
- 명칭 통일: 홈 배너·데일리·CLI act 모두 **지금 할 일**.

## 하루 요약 (Executive Summary)

- `EXEC_SUMMARY` 프롬프트 + `ai_exec_summary()` — 파이프라인 **마지막**(분류·정제
  후)에 sonnet 1콜. 최종 큐 기준으로 쓴다. 단계 표시는 5→6단계.
- **입력은 이미 추출·검증된 한 줄들뿐**(델타·할 일·발신 목록·흐름·수신 수) — 원문
  없음 → 토큰 바운드(~2KB), 환각 표면 최소. "입력에 없는 사실·번호 금지, (#번호)
  인용" 규칙.
- '내 활동'의 핵심 팩트는 AI 추측이 아니라 데이터: `Store.action_closed_by_me_on(d)`
  가 fold_action **재생**으로 "그날 내 실질 발신이 열린 액션 슬롯을 종결시킨 스레드"
  를 계산한다(스키마 추가 없음 — 대상이 그날 발신 스레드뿐이라 비용 미미. 이후 새
  요청으로 다시 열려도 그날 종결 사실은 유지).
- graceful: 백엔드 미설정·실패 → None → render 가 결정론 한 줄로 대체
  (`수신 19 · 발신 2 · 할 일 3 — 최우선: [#111] …`). 결정론 리뷰 생존 원칙(#10) 유지.

## 웹 렌더 규칙 (md는 평문 유지, 렌더러만 보강)

- 첫 `##` 이전의 문단들 → 요약 카드 `.dsum` (구조 기준 — 이름 아님).
- `## 참고` → `<details class='dref'>` 접힘.
- 인라인 장식: `[상|중|하]` → 색 칩, `★` → 골드, `⏰기한`·`⚠` → 경고색,
  `「…」` → 흐림. 데일리 전용 `_md_inline` — 메일 본문(`_mail_md_*`)과 무관.
- 들여쓴 비-불릿 줄(↳ 사유·「근거」)은 열린 `<li>`의 연속 줄(lazy continuation) —
  목록을 끊지 않는다.
- **옛 형식 데일리 호환**: 머리 문단·'참고' 헤딩이 없으므로 카드·접힘 미적용,
  기존 렌더 그대로. `[상]`·★ 장식은 옛 파일에도 적용(개선).

## UI 후속 정리 (2026-07-17, 사용자 요청)

- **최상위 메뉴 `기록` → `기억`.** '기록'은 앱의 모든 것(메일·스레드·노트)에
  해당해 변별력이 없었다. 확정 용어 '장기기억'과 한 체계 — 하루 단위 기억(데일리)
  ↔ 영구 기억(장기기억). 경로 `/records` 는 유지(URL↔표시명 불일치는 무해하고,
  바꾸면 옛 북마크 호환 분기만 늘어난다 — `/daily` 흡수 전례와 동일한 판단).
  '기록만 남기기' 버튼은 메뉴와 무관한 동작 이름이라 그대로.
- **홈 'AI 정리' 입력창·버튼 제거** (`/refine`·구 `/lens/intervene/refine` 라우트,
  `.refine` CSS, app.js 경로 매핑 동반 삭제). 실사용이 없었고, `PROPOSAL-distill.md`
  가 Phase 2 예고분("자동 주석은 유지")으로 남겨둔 정리를 이행한 것.
  **`ai_refine_intervention` 자체는 유지** — '오늘 메일 정리'(run_ai_layer)와 CLI
  `act --ai` 가 계속 호출하고, 홈은 저장된 주석을 `apply_saved_ai` 로 읽어 반영한다.
  즉 없어진 것은 '홈에서 메모를 주고 재실행하는 경로'뿐이다.
